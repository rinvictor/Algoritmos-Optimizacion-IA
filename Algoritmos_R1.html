def fibonacci_iter(n):
    fib = [0, 1]
    
    for i in range(2, n + 1):
        fib.append(fib[i-1] + fib[i-2])

    return fib[n]

Conclusiones

Como vemos con la primera aproximacin, utilizando un enfoque recursivo, el algoritmo tiene un coste computacional que crece de manera exponencial con una complejidad de O(2^n). Esto se debe a la cantidad de operaciones redundantes que se deben hacer. Si en lugar de utilizar un algoritmo basado en recursividad se usa un algoritmo iterativo el crecimiento es lineal O(n), el tiempo de cmputo crece de manera directamente proporcional a la entrada. Elegir una aproximacin iterativa para implementar la sucesin de fibonacci mejora significativamente la eficiencia del algoritmo.

def cambio_moneda(cantidad_ingresada_cnts):
    resultado = []
    tipo_monedas.sort(reverse=True)
    for moneda in tipo_monedas:
        while cantidad_ingresada_cnts >= moneda:
            resultado.append(moneda)
            cantidad_ingresada_cnts = cantidad_ingresada_cnts - moneda
    return resultado

N-Reinas por tcnica de vuelta atrs

Resuelve el Problema de las N-Reinas en un tablero de dimensin N mediante la tcnica de la vuelta atrs (backtraking).

El problema de las N-reinas consiste en colocar N reinas en un tablero de NxN sin se que se amenacen entre s. Las reinas se amenazan en el caso de estar en la misma linea, fila o diagonal. Para N=2 y N=3 el problema no tiene solucin.

In[]:
def es_seguro(tablero, fila, col):
    for i in range(fila):
        if tablero[i] == col or tablero[i] - i == col - fila or tablero[i] + i == col + fila:
            return False
    return True

def colocar_reinas(tablero, fila, n):
    if fila == n: # Se llega a una solucin
        print(tablero)
        return

    for col in range(n):
        if es_seguro(tablero, fila, col):
            tablero[fila] = col
            colocar_reinas(tablero, fila + 1, n)
            # backtrack
            tablero[fila] = -1
